# Contributing

## Общие правила

- Основной язык общения в проекте: русский.
- Изменения держать небольшими и предметными.
- Не вносить breaking changes в публичный API без отдельного обсуждения.
- Если меняется публичный API, обновлять типы, exports, README и примеры.

## Подготовка окружения

```bash
yarn install
```

Основные проверки:

```bash
yarn build
yarn typecheck
yarn lint
yarn format
```

## Стиль изменений

- Следовать форматированию из `.editorconfig` и `.prettierrc`.
- Предпочитать строгую типизацию TypeScript.
- Не добавлять новые зависимости без явной необходимости.
- Если логика неочевидна, добавлять короткие комментарии по месту.

## Документация и примеры

- Для пользовательских изменений обновлять README, если меняется поведение или способ использования.
- Для новых API и сценариев по возможности добавлять или актуализировать примеры в `examples/`.

## Pull Request

Перед отправкой PR желательно:

- убедиться, что изменения локально собираются;
- прогнать релевантные проверки;
- кратко описать цель изменения, затронутые модули и возможные риски регрессии.

## Лицензия вкладов

Отправляя вклад в репозиторий, вы соглашаетесь, что он будет распространяться
по лицензии MIT этого проекта.
